# Feature verification — pre-1.0

This is the single tracking thread for the pre-1.0 feature verification effort. The goal: confirm that **every** AI Boost for Joomla option does what it claims — and that it does so **cleanly on all installed languages** — before we cut the 1.0 release. "Cleanly" means: the setting saves, its runtime consumer fires, the expected artefact appears in the right place (head / body / robots.txt / sitemap / file / DB), Pro gating behaves, and on multilingual sites the per-language overlay is correct (no fallback to the default language where a translation exists, no missing-translation crashes).

**How to use the Status column:** every row starts as `waiting`. As each feature is verified on staging (all-on test config + active Pro, inspected per language in a subagent), flip its Status to `works`, `broken`, or `partial`, and record which installed languages it was checked against in the Languages column. This file spans sessions — pick up wherever the Status/Languages columns left off; do not re-verify rows already marked `works` unless the underlying code changed.

**Legend**
- **Status:** `waiting` (not yet checked) · `works` (verified correct) · `broken` (verified failing) · `partial` (some paths/languages work, others don't — explain in Notes).
- **Languages:** the installed languages the row was verified on; starts at `-` (nothing verified yet). E.g. `en-GB, de-DE, sr-RS`.
